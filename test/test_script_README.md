# LLM部署系统测试脚本使用说明

## 脚本概述

本测试脚本用于在远程测试环境中对LLM部署系统进行系统性测试，评估系统的功能完整性、性能表现和稳定性。脚本设计为仅执行测试准备和测试流程。

## 测试覆盖范围

脚本包含以下测试模块：

1. **环境检查模块**：验证测试环境的依赖项、硬件资源和模型文件完整性
2. **模型加载/卸载测试**：确保每个模型都能正确加载/卸载，卸载时释放相关资源
3. **基础功能测试**：验证文本生成、问答交互等核心功能
4. **模型类型专用测试**：针对文本模型、多模态模型、嵌入模型设计专用测试场景
5. **性能基准测试**：记录模型响应时间、内存占用等关键指标

## 支持的模型

脚本支持测试以下模型：

- **文本模型**：
  - Qwen/Qwen3-8B
  - Qwen/Qwen3-30B-A3B-Instruct-2507

- **多模态模型**：
  - Qwen/Qwen3-VL-8B-Thinking

- **嵌入模型**：
  - BAAI/bge-large-zh-v1.5
  - BAAI/bge-m3

## 使用方法

### 基本用法

```bash
python test_script.py
```

此命令将执行完整的测试套件，包括所有测试模块，并生成测试报告。

### 参数说明

脚本支持以下命令行参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--api-url` | API服务地址 | `http://127.0.0.1:8000` |
| `--model-dir` | 模型权重目录 | 从环境变量 `MODEL_WEIGHTS_DIR` 获取，或默认 `/data2/home/songzhihua/LLM/llm_playground/model_weights` |
| `--output` | 测试报告输出文件 | `test_report.json` |
| `--depth` | 测试深度 (1-3，数字越大测试越全面) | `1` |
| `--models` | 指定要测试的模型列表 | 测试所有支持的模型 |
| `--test-env` | 执行环境检查测试 | `True` |
| `--test-load` | 执行模型加载/卸载测试 | `True` |
| `--test-basic` | 执行基础功能测试 | `True` |
| `--test-specific` | 执行模型类型专用测试 | `True` |
| `--test-performance` | 执行性能基准测试 | `True` |

### 示例用法

1. **执行特定深度的测试**：
   ```bash
   python test_script.py --depth 2
   ```

2. **测试特定模型**：
   ```bash
   python test_script.py --models Qwen/Qwen3-8B BAAI/bge-large-zh-v1.5
   ```

3. **跳过某些测试模块**：
   ```bash
   python test_script.py --test-performance False
   ```

4. **自定义API地址和输出文件**：
   ```bash
   python test_script.py --api-url http://192.168.1.100:8000 --output my_test_report.json
   ```

## 测试报告

脚本执行完成后，会生成以下测试报告：

1. **简洁JSON报告** (`test_report/test_report.json`)：包含测试结果的摘要信息
2. **详细JSON报告** (`test_report/test_report_detailed.json`)：包含完整的测试结果和性能数据
3. **HTML报告** (`test_report/test_report.html`)：美观的可视化测试报告

### HTML报告示例

HTML报告包含以下内容：
- 测试摘要：总测试数、通过数、失败数、成功率
- 环境检查结果
- 模型加载/卸载测试结果
- 基础功能测试结果
- 模型类型专用测试结果
- 性能基准测试结果
- 详细的错误信息（如果有）

## 环境要求

### 运行环境

- Python 3.8+
- 必要的Python包：
  - requests
  - psutil
  - torch
  - transformers
  - FlagEmbedding

### 硬件要求

- 内存：至少16GB（测试大型模型时建议32GB以上）
- GPU：测试多模态模型时需要支持CUDA的GPU
- 存储空间：足够存储测试模型的权重文件

## 注意事项

1. **测试时间**：完整测试套件可能需要较长时间执行，特别是测试大型模型时
2. **资源占用**：测试过程中会加载和卸载模型，可能会占用大量GPU内存
3. **网络连接**：确保测试环境能够正常访问API服务
4. **模型文件**：确保模型文件已正确放置在指定目录
5. **权限**：确保脚本具有执行权限和访问模型文件的权限

## 故障排除

### 常见问题

1. **API连接失败**：检查API服务是否正在运行，以及网络连接是否正常
2. **模型加载失败**：检查模型文件是否完整，以及硬件资源是否足够
3. **测试超时**：对于大型模型，可能需要增加超时时间
4. **内存不足**：测试大型模型时，确保系统有足够的内存

### 日志文件

脚本执行过程中会生成日志文件 `test_results.log`，包含详细的执行信息和错误日志，可用于故障排除。

## 扩展和定制

### 添加新的测试用例

可以通过修改以下函数添加新的测试用例：

- `test_text_generation`：添加文本生成测试用例
- `test_qa_interaction`：添加问答交互测试用例
- `test_model_specific_scenarios`：添加模型类型专用测试用例

### 支持新的模型

要支持新的模型，需要：

1. 在 `TEST_MODELS` 字典中添加新模型
2. 确保模型文件已正确放置在模型权重目录中
3. 根据需要为新模型添加专用测试用例
